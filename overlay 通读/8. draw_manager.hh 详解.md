## 概述 
[[⬆](#目录)]

`draw_manager.hh` 是Blender绘制系统的核心文件，定义了 `Manager` 类，该类作为场景数据和视口引擎之间的接口。Manager类负责管理每个组件的数据（ObjectInfo、ObjectMatrices等），并通过ResourceHandle进行索引。

## 2. 核心组件分析

### 2.1. Manager类的核心地位 
[[⬆](#目录)]

#### 2.1.1. 在绘制系统中的作用

```cpp
// source/blender/draw/intern/draw_manager.hh:10-15
/**
 * `draw::Manager` is the interface between scene data and viewport engines.
 *
 * It holds per component data (`ObjectInfo`, `ObjectMatrices`, ...) indexed per `ResourceHandle`.
 *
 * \note It is currently work in progress and should replace the old global draw manager.
 */
```

Manager类在Blender绘制系统中扮演以下关键角色：

1. **数据管理中心**：统一管理所有绘制相关的对象数据
2. **资源分配器**：为每个对象分配唯一的ResourceHandle
3. **GPU资源协调者**：管理GPU缓冲区的分配和同步
4. **可见性计算器**：执行对象的视锥剔除和可见性判断

#### 2.1.2. 与其他系统的关系

```cpp
// source/blender/draw/intern/draw_manager.hh:23-24
#include "draw_resource.hh"
#include "draw_view.hh"
```

Manager类与以下系统紧密协作：

- **draw_resource.hh**：提供ObjectMatrices、ObjectInfos等数据结构
- **draw_view.hh**：提供View类进行可见性计算
- **GPU系统**：通过GPU缓冲区进行数据传输
- **材质系统**：通过GPUMaterial处理着色器属性

#### 2.3.1. 缓冲区类型定义

```cpp

  using ObjectMatricesBuf = StorageArrayBuffer<ObjectMatrices, 128>;
  using ObjectBoundsBuf = StorageArrayBuffer<ObjectBounds, 128>;

**缓冲区类型详解**：

1. **StorageArrayBuffer**：存储数组缓冲区，用于大量对象数据
   - 模板参数：数据类型，缓冲区大小（128）
   - 适用于：对象矩阵、边界、信息等频繁访问的数据

2. **UniformArrayBuffer**：统一数组缓冲区，用于着色器访问
   - 模板参数：数据类型，缓冲区大小（512）
   - 适用于：层属性等需要着色器访问的数据

inline ResourceHandleRange Manager::unique_handle(const ObjectRef &ref)
{
  if (!ref.handle_.is_valid()) {
    /* WORKAROUND: Instead of breaking const correctness everywhere, we only break it for this. */
    const_cast<ObjectRef &>(ref).handle_ = resource_handle(ref);
  }
  return ref.handle_;
}
```

**设计原理**：
- 检查对象是否已有有效句柄
- 如果没有，创建新句柄并赋值（通过const_cast绕过const限制）
- 返回现有或新创建的句柄范围

#### 3.1.2. 对象资源句柄创建

```cpp
// source/blender/draw/intern/draw_manager.hh:324-361
inline ResourceHandleRange Manager::resource_handle(const ObjectRef &ref, float inflate_bounds)
{
  bool is_active_object = ref.is_active(object_active);
  bool is_active_edit_mode = object_active &&
                             (DRW_object_is_in_edit_mode(object_active) ||
                              ELEM(object_active->mode, OB_MODE_TEXTURE_PAINT, OB_MODE_SCULPT)) &&
                             ref.object->mode == object_active->mode;
  if (ref.duplis_) {
    uint start = resource_len_;

    ObjectBounds proto_bounds;
    proto_bounds.sync(*ref.object, inflate_bounds);

    ObjectInfos proto_info;
    proto_info.sync(ref, is_active_object, is_active_edit_mode);

    for (const DupliObject *dupli : *ref.duplis_) {
      matrix_buf.current().get_or_resize(resource_len_).sync(float4x4(dupli->mat));
      bounds_buf.current().get_or_resize(resource_len_) = proto_bounds;

      ObjectInfos &info = infos_buf.current().get_or_resize(resource_len_);
      info = proto_info;
      info.random = dupli->random_id * (1.0f / (float)0xFFFFFFFF);

      resource_len_++;
    }
    return ResourceHandleRange(ResourceHandle(start, (ref.object->transflag & OB_NEG_SCALE) != 0),
                               resource_len_ - start);
  }
  else {
    matrix_buf.current().get_or_resize(resource_len_).sync(*ref.object);
    bounds_buf.current().get_or_resize(resource_len_).sync(*ref.object, inflate_bounds);
    infos_buf.current()
        .get_or_resize(resource_len_)
        .sync(ref, is_active_object, is_active_edit_mode);
    return ResourceHandle(resource_len_++, (ref.object->transflag & OB_NEG_SCALE) != 0);
  }
}
```

**实现细节**：

1. **状态判断**：
   - `is_active_object`：是否为活动对象
   - `is_active_edit_mode`：是否处于活动编辑模式

2. **重复对象处理**：
   - 创建原型边界和信息对象
   - 为每个重复对象复制数据并设置随机ID
   - 返回句柄范围

3. **单个对象处理**：
   - 直接同步对象数据到缓冲区
   - 返回单个资源句柄

### 3.2. 对象属性提取 
[[⬆](#目录)]

```cpp
// source/blender/draw/intern/draw_manager.hh:434-452
inline void Manager::extract_object_attributes(ResourceHandle handle,
                                               const ObjectRef &ref,
                                               const GPUMaterial *material)
{
  ObjectInfos &infos = infos_buf.current().get_or_resize(handle.resource_index());
  infos.object_attrs_offset = attribute_len_;

  const GPUUniformAttrList *attr_list = GPU_material_uniform_attributes(material);
  if (attr_list == nullptr) {
    return;
  }

  LISTBASE_FOREACH (const GPUUniformAttr *, attr, &attr_list->list) {
    if (attributes_buf.get_or_resize(attribute_len_).sync(ref, *attr)) {
      infos.object_attrs_len++;
      attribute_len_++;
    }
  }
}
```

**属性提取流程**：

1. **设置属性偏移**：记录当前属性缓冲区位置
2. **获取材质属性列表**：从GPU材质中提取统一属性
3. **遍历属性列表**：为每个属性调用sync方法
4. **更新计数器**：成功同步后增加属性长度和计数

### 5.1. ResourceHandle设计 
[[⬆](#目录)]

```cpp
// source/blender/draw/intern/draw_handle.hh:114-146
class ResourceHandle {
  friend class Manager;
  friend class ResourceHandleRange;

  ResourceIndex index_ = {};

  ResourceHandle(uint raw) : index_(raw) {}
  ResourceHandle(uint index, bool inverted_handedness) : index_(index, inverted_handedness) {}

public:
  ResourceHandle() = default;

  bool is_valid() const
  {
    return index_.raw != 0;
  }

  bool has_inverted_handedness() const
  {
    return index_.has_inverted_handedness();
  }

  uint resource_index() const
  {
    return index_.resource_index();
  }
};
```

**设计特点**：
1. **安全性**：只能由Manager类创建有效句柄
2. **紧凑性**：使用32位整数编码索引和手性信息
3. **类型安全**：通过友元类控制访问权限

### 5.2. ResourceIndex编码 
[[⬆](#目录)]

```cpp
// source/blender/draw/intern/draw_handle.hh:56-76
struct ResourceIndex {
  uint32_t raw;

  ResourceIndex() = default;
  ResourceIndex(uint raw_) : raw(raw_) {};
  ResourceIndex(uint index, bool inverted_handedness)
  {
    raw = index;
    SET_FLAG_FROM_TEST(raw, inverted_handedness, 0x80000000u);
  }

  bool has_inverted_handedness() const
  {
    return (raw & 0x80000000u) != 0;
  }

  uint resource_index() const
  {
    return (raw & 0x7FFFFFFFu);
  }
};
```

**编码方案**：
- 最高位（31位）：手性标志（负缩放）
- 低31位：资源索引
- 支持最多21亿个资源句柄

## 8. 总结 
[[⬆](#目录)]

Manager类是Blender现代绘制系统的核心组件，具有以下关键特性：

#### 8.1. 设计优势

1. **统一管理**：集中管理所有绘制相关数据
2. **高性能**：通过双缓冲、批处理等技术优化性能
3. **类型安全**：强类型资源句柄系统
4. **可扩展**：支持多种绘制通道和视图类型

#### 8.2. 核心创新

1. **ResourceHandle系统**：紧凑高效的资源索引方案
2. **SwapChain缓冲区**：避免GPU同步等待
3. **延迟计算**：可见性和命令生成的按需计算
4. **属性间接引用**：灵活的对象属性管理

#### 8.3. 应用场景

Manager类广泛应用于：
- EEVEE渲染引擎
- Workbench渲染引擎  
- Overlay绘制引擎
- 几何体绘制系统

这个设计为Blender提供了高性能、可维护的绘制基础设施，是现代GPU渲染架构的优秀实现。
