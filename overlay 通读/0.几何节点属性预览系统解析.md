## 概述

几何节点属性预览系统是Blender overlay引擎中的一个重要组件，用于在3D视图中实时显示几何节点计算出的属性值。该系统通过颜色映射和文本标注两种方式，将抽象的属性数据可视化，帮助用户理解和调试几何节点网络。

## 系统架构 [[⬆](#目录)]

### 1. 核心组件

几何节点属性预览系统主要由以下几个核心类组成：

- **AttributeViewer类** (`overlay_attribute_viewer.hh:26`): 主要的颜色预览渲染器
- **AttributeTexts类** (`overlay_attribute_text.hh:41`): 文本标注渲染器
- **着色器系统**: 处理不同几何类型的属性可视化

### 2. 数据流程

```
几何节点计算 → .viewer属性 → 属性预览系统 → 3D视图显示
```

1. 几何节点将结果存储在`.viewer`属性中
2. AttributeViewer和AttributeTexts检测到`.viewer`属性
3. 根据几何类型选择相应的渲染策略
4. 通过着色器将属性值转换为可视化效果

### 2. 初始化过程

#### begin_sync函数 (`overlay_attribute_viewer.hh:37`)

```cpp
void begin_sync(Resources &res, const State &state) final
{
  ps_.init();
  enabled_ = state.is_space_v3d() && !res.is_selection() && state.show_attribute_viewer();
  if (!enabled_) {
    return;
  };
  ps_.bind_ubo(OVERLAY_GLOBALS_SLOT, &res.globals_buf);
  ps_.bind_ubo(DRW_CLIPPING_UBO_SLOT, &res.clip_planes_buf);
  ps_.state_set(DRW_STATE_WRITE_COLOR | DRW_STATE_DEPTH_LESS_EQUAL | DRW_STATE_BLEND_ALPHA,
                state.clipping_plane_count);
```

**关键点分析**：
- **启用条件**: 只在3D视图空间、非选择模式、且启用属性查看器时工作
- **深度测试设置**: `DRW_STATE_DEPTH_LESS_EQUAL` (第46行) - 这是导致遮挡问题的关键
- **混合模式**: `DRW_STATE_BLEND_ALPHA` - 支持透明度混合

  }

**处理特点**：

#### 3.2 点云(PointCloud)处理 (`overlay_attribute_viewer.hh:185`)

```cpp
case OB_POINTCLOUD: {
  PointCloud &pointcloud = DRW_object_get_data_for_drawing<PointCloud>(object);
  if (const std::optional<bke::AttributeMetaData> meta_data =
          pointcloud.attributes().lookup_meta_data(".viewer"))
  {
    if (attribute_type_supports_viewer_overlay(meta_data->data_type)) {
      gpu::VertBuf **vertbuf = DRW_pointcloud_evaluated_attribute(&pointcloud, ".viewer");
      if (pointcloud.totpoint > 0 && vertbuf != nullptr) {
        auto &sub = *pointcloud_sub_;
        gpu::Batch *batch = pointcloud_sub_pass_setup(sub, &object, nullptr);
        sub.push_constant("opacity", opacity);
        sub.bind_texture("attribute_tx", vertbuf);
        sub.draw(batch, manager.unique_handle(ob_ref));
      }
    }
  }
  break;
}
```

**处理特点**：
- 通过纹理缓冲区传递属性数据
- 使用`attribute_tx`纹理采样器
- 需要检查点云数据有效性

#### 3.3 曲线(Curves)处理 (`overlay_attribute_viewer.hh:222`)

```cpp
case OB_CURVES: {
  ::Curves &curves_id = DRW_object_get_data_for_drawing<::Curves>(object);
  const bke::CurvesGeometry &curves = curves_id.geometry.wrap();
  if (const std::optional<bke::AttributeMetaData> meta_data =
          curves.attributes().lookup_meta_data(".viewer"))
  {
    if (attribute_type_supports_viewer_overlay(meta_data->data_type)) {
      bool is_point_domain;
      bool is_valid;
      gpu::VertBufPtr &texture = DRW_curves_texture_for_evaluated_attribute(
          &curves_id, ".viewer", is_point_domain, is_valid);
      if (is_valid) {
        auto &sub = *curves_sub_;
        const char *error = nullptr;
        gpu::Batch *batch = curves_sub_pass_setup(sub, state.scene, ob_ref.object, error);
        sub.push_constant("opacity", opacity);
        sub.push_constant("is_point_domain", is_point_domain);
        sub.bind_texture("color_tx", texture);
        sub.draw(batch, manager.unique_handle(ob_ref));
      }
    }
  }
  break;
}
```

**处理特点**：
- 区分点域和曲线域属性
- 使用`is_point_domain`标志控制着色器行为
- 通过`color_tx`纹理传递属性数据

## 深度测试行为分析 [[⬆](#目录)]

### 1. 当前深度测试设置

在`overlay_attribute_viewer.hh:46`行：
```cpp
ps_.state_set(DRW_STATE_WRITE_COLOR | DRW_STATE_DEPTH_LESS_EQUAL | DRW_STATE_BLEND_ALPHA,
              state.clipping_plane_count);
```

**DRW_STATE_DEPTH_LESS_EQUAL的含义**：
- 只绘制深度值小于等于当前深度缓冲值的片段
- 这意味着被其他物体遮挡的部分不会被显示
- 符合正常的3D渲染深度测试规则

### 2. 深度测试问题分析

**问题描述**：
当几何节点属性预览被其他物体遮挡时，被遮挡部分完全不可见，用户无法看到完整的属性分布。

**根本原因**：
`DRW_STATE_DEPTH_LESS_EQUAL`状态遵循标准的深度测试规则，只显示未被遮挡的表面。

## 属性域适配问题 [[⬆](#目录)]

### 1. 属性域类型

Blender支持多种属性域：
- **点域(Point Domain)**: 每个顶点/点一个属性值
- **边域(Edge Domain)**: 每条边一个属性值  
- **面域(Face Domain)**: 每个面一个属性值
- **角域(Corner Domain)**: 每个面角一个属性值
- **实例域(Instance Domain)**: 每个实例一个属性值

### 2. 当前域处理机制

#### 2.1 网格域处理
当前系统主要通过`DRW_cache_mesh_surface_viewer_attribute_get`获取预览数据，但这个函数主要处理面域和点域属性。
