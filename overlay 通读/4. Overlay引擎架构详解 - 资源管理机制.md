## 概述 [[⬆](#目录)]

Overlay引擎的资源管理机制是其高性能渲染的核心，通过精细的GPU资源管理、缓冲区优化和内存池化技术，实现了高效的实时渲染。本文档深入分析Overlay引擎的资源管理系统，包括GPU资源创建、缓冲区管理、纹理管理、着色器管理以及性能优化策略。

## 2. 资源管理系统架构 [[⬆](#目录)]

### 2.1. 核心管理类

Overlay引擎的资源管理基于以下几个核心类：

```cpp
// source/blender/draw/intern/draw_manager.hh:45-97
class Manager {
  using ObjectMatricesBuf = StorageArrayBuffer<ObjectMatrices, 128>;
  using ObjectBoundsBuf = StorageArrayBuffer<ObjectBounds, 128>;
  using ObjectInfosBuf = StorageArrayBuffer<ObjectInfos, 128>;
  using ObjectAttributeBuf = StorageArrayBuffer<ObjectAttribute, 128>;
  using LayerAttributeBuf = UniformArrayBuffer<LayerAttribute, 512>;

public:
  /**
   * 包含所有对象数据的缓冲区，通过资源索引引用
   * 在同步后暴露为公共成员供着色器访问
   */
  SwapChain<ObjectMatricesBuf, 2> matrix_buf;
  SwapChain<ObjectBoundsBuf, 2> bounds_buf;
  SwapChain<ObjectInfosBuf, 2> infos_buf;

  /**
   * 对象属性通过ObjectInfos中的间接数据引用
   * 这是因为属性列表是任意的
   */
  ObjectAttributeBuf attributes_buf;

  /**
   * 着色器所需的所有视图层属性表，用于填充下面的缓冲区
   */
  Map<uint32_t, GPULayerAttr> layer_attributes;

  /**
   * 按哈希值索引和排序的层属性值缓冲区
   */
  LayerAttributeBuf layer_attributes_buf;

  /**
   * 来自Image数据块的纹理列表
   * 需要进行引用计数以避免在其他线程中被释放
   */
  Vector<gpu::Texture *> acquired_textures;
};
```

### 7.2. 资源更新和同步

资源同步确保GPU数据的一致性：

```cpp
// source/blender/draw/intern/draw_manager.hh:434-483
inline void Manager::extract_object_attributes(ResourceHandle handle,
                                       const ObjectRef &ref,
                                       Span<GPUMaterial *> materials)
{
  ObjectInfos &infos = infos_buf.current().get_or_resize(handle.resource_index());
  infos.object_attrs_offset = attribute_len_;

  /* 简单的缓存解决方案以避免重复 */
  Vector<uint32_t, 4> hash_cache;

  for (const GPUMaterial *mat : materials) {
    const GPUUniformAttrList *attr_list = GPU_material_uniform_attributes(mat);
    if (attr_list == nullptr) {
      continue;
    }

    LISTBASE_FOREACH (const GPUUniformAttr *, attr, &attr_list->list) {
      /** 线性搜索。避免跨材质的重复属性 */
      if ((mat != materials.first()) && 
          (hash_cache.first_index_of_try(attr->hash_code) != -1)) {
        /* 属性已被另一个材质添加到属性缓冲区 */
        continue;
      }
      hash_cache.append(attr->hash_code);
      if (attributes_buf.get_or_resize(attribute_len_).sync(ref, *attr)) {
        infos.object_attrs_len++;
        attribute_len_++;
      }
    }
  }
}
```

struct ShapeInstanceBuf : private select::SelectBuf {
  StorageVectorBuffer<InstanceDataT> data_buf;

      : select::SelectBuf(selection_type), data_buf(name) {};

## 11. 总结 [[⬆](#目录)]

Overlay引擎的资源管理机制通过以下关键技术实现了高效的实时渲染：

### 10.1 核心优势

1. **分层管理**：通过Manager、ShaderModule、Resources等多层管理实现清晰的职责分离
2. **池化技术**：使用对象池、纹理池、着色器缓存减少创建/销毁开销
3. **延迟更新**：批量更新和延迟同步减少GPU状态切换
4. **内存优化**：精确的对齐、填充和布局优化减少内存浪费
5. **引用计数**：智能的引用计数机制防止资源泄漏

### 10.2 性能特征

1. **低开销**：资源创建和销毁最小化
2. **高效率**：批量操作和状态缓存减少GPU调用
3. **可扩展**：模块化设计支持功能扩展
4. **稳定性**：完善的错误处理和调试支持

### 10.3 设计模式

1. **RAII**：自动资源管理
2. **工厂模式**：统一的资源创建接口
3. **缓存模式**：多级缓存提升性能
4. **策略模式**：灵活的着色器变体管理

这种精心设计的资源管理机制使Overlay引擎能够在复杂的3D场景中实现流畅的实时渲染，同时保持较低的内存占用和GPU负载。
