# 14. draw_manager_text.cc 详解

## 文本渲染系统概述

### 文本渲染在overlay中的作用
文本渲染是Blender 3D视口中不可或缺的功能，主要用于：
- **测量信息显示**：边长、角度、面积等几何测量数据
- **调试信息显示**：顶点、边、面的索引信息
- **编辑模式反馈**：实时显示编辑操作的数值结果
- **用户界面元素**：3D空间中的文本标注

### 文本渲染的挑战和解决方案

#### 主要挑战
1. **性能问题**：大量文本同时渲染时的性能瓶颈
2. **空间定位**：3D坐标到2D屏幕坐标的精确转换
3. **视口裁剪**：文本在视口边界外的正确处理
4. **字体渲染**：不同分辨率下的清晰度保持
5. **内存管理**：动态文本数据的内存分配和释放

#### 解决方案
1. **批处理缓存**：使用`BLI_memiter`实现高效的文本缓存
2. **延迟渲染**：先收集所有文本，统一渲染
3. **智能裁剪**：3D空间裁剪避免不必要的渲染
4. **GPU加速**：利用GPU进行文本渲染
5. **内存池**：预分配内存块减少动态分配

### 性能考虑
- **缓存机制**：避免重复的字符串拷贝和内存分配
- **状态切换最小化**：批量处理相同颜色的文本
- **视口裁剪**：提前剔除不可见的文本
- **字符串指针优化**：支持指针引用避免字符串拷贝

## 文本渲染流程

### 完整渲染流程图
```
文本添加阶段:
1. DRW_text_cache_add() 
   ├── 计算内存需求
   ├── 分配ViewCachedString
   ├── 填充文本数据
   └── 存储到memiter

投影阶段:
2. DRW_text_cache_draw()
   ├── 3D坐标投影到屏幕
   ├── 视口裁剪检测
   └── 标记可见文本

渲染阶段:
3. drw_text_cache_draw_ex()
   ├── 设置GPU状态
   ├── 批量处理相同颜色
   ├── 字符串测量和对齐
   └── BLF绘制调用
```

### 字符网格生成过程
虽然文本渲染主要使用BLF库，但内部涉及字符网格生成：

1. **字体加载**：BLF从系统加载字体文件
2. **字符光栅化**：将矢量字体转换为位图
3. **纹理生成**：字符位图上传到GPU纹理
4. **网格构建**：为每个字符生成四边形网格
5. **UV映射**：将字符网格映射到字体纹理

## 总结

`draw_manager_text.cc`实现了一个高效、灵活的3D文本渲染系统，其核心特点包括：

### 技术亮点
1. **高效缓存**：`BLI_memiter` + 柔性数组实现零拷贝文本存储
2. **智能投影**：支持3D到2D的精确坐标转换和裁剪
3. **批处理优化**：颜色状态批处理减少GPU切换
4. **单位系统**：完整的单位转换和本地化支持

### 设计模式
1. **延迟渲染**：收集-投影-渲染的三阶段模式
2. **策略模式**：不同测量类型的统一处理接口
3. **对象池**：内存预分配和复用机制

### 性能特征
- **时间复杂度**：O(n)线性遍历，常数时间插入
- **空间复杂度**：紧凑内存布局，最小化内存占用
- **GPU友好**：批处理减少状态切换，提高渲染效率

这个文本渲染系统为Blender的3D编辑体验提供了重要支撑，在保证性能的同时实现了丰富的文本显示功能。
