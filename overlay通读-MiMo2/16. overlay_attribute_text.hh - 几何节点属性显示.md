### 3. 特殊处理: Corner域文本

**问题**: 在面上显示属性文本时需要避免重叠。

**解决方案**: `calc_corner_text_position()` (Lines 366-400)

```cpp
/* 计算文本位置 */
float3 calc_corner_text_position(
  const float3 &corner_pos,      // 角点位置
  const float3 &prev_corner_pos, // 上一个角点
  const float3 &next_corner_pos, // 下一个角点
  const float3 &face_normal,     // 面法线
  const RegionView3D *rv3d,      // 视图信息
  const float4x4 &object_to_world,
  const float offset_scale = 1.0f)
{
  /* 1. 计算两条边的方向 */
  prev_edge_vec = prev_corner_pos - corner_pos;
  next_edge_vec = next_corner_pos - corner_pos;
  prev_edge_dir = normalize(prev_edge_vec);
  next_edge_dir = normalize(next_edge_vec);

  /* 2. 计算角平分线 (偏移方向) */
  concavity_check = dot(cross(next_edge_dir, prev_edge_dir), face_normal);
  direction_correct = concavity_check > 0.0f ? 1.0f : -1.0f;
  bisector_dir = (prev_edge_dir + next_edge_dir) / 2 * direction_correct;

  /* 3. 根据角度调整偏移距离 (尖锐角落偏移更大) */
  sharp_factor = clamp(dot(prev_edge_dir, next_edge_dir), 0.0f, 1.0f);
  sharp_multiplier = pow(sharp_factor, 4.0f) * 2 + 1;  // 1.0 ~ 3.0

  /* 4. 像素到世界空间的转换 */
  float3 pos_in_world = transform_point(object_to_world, corner_pos);
  pixel_size = ED_view3d_pixel_size(rv3d, pos_in_world);
  screen_space_offset = pixel_size * widget_points * 7.0f;
  offset_distance = clamp(screen_space_offset * sharp_multiplier * offset_scale,
                          0.0f, max_offset);

  /* 5. 最终位置 */
  return corner_pos + bisector_dir * offset_distance;
}
```

**数学原理**:
```
偏移量 = 像素偏移 × 角度修正系数 × 缩放因子
距离限制 = min(相邻边长) / 2  (防止文本过大)
```

## 坐标计算流程

```
原始位置
  ↓
对象到世界变换 (object_to_world)
  ↓
角域特殊处理 (corner domain only)
  └─> 计算边向量
  └─> 计算角平分线
  └─> 根据像素偏移调整位置
  └─> 返回世界空间偏移坐标
  ↓
DRW_text_cache_add (添加到缓存)
  ↓
后续由 draw_manager_text.cc 处理：
  - 3D投影到2D
  - 可见性测试
  - 实际绘制
```

## 性能考虑

1. **并行计算**: Corner域使用`threading::parallel_for`处理大量面
2. **类型特化**: C++模板在编译期生成转换代码
3. **懒计算**: 仅在检查到`.viewer`属性时才处理
4. **内存复用**: 使用`state.dt`统一文本缓存
