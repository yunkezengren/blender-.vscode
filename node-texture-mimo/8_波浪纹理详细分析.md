## 概述

波浪纹理节点是Blender中用于生成程序化波浪图案的核心节点。它支持两种波形类型（Bands/Rings）和三种剖面形状（Sine/Saw/Tri），并提供完整的扭曲和细节控制。

## 波形类型数学原理

### 类型1: Bands (带状波)

**数学原理**: 沿单一轴向的线性波浪

```cpp
// C++实现 (lines 162-178)
switch (bands_direction_) {
  case SHD_WAVE_BANDS_DIRECTION_X:
    n = p.x * 20.0f;  // X轴方向
    break;
  case SHD_WAVE_BANDS_DIRECTION_Y:
    n = p.y * 20.0f;  // Y轴方向
    break;
  case SHD_WAVE_BANDS_DIRECTION_Z:
    n = p.z * 20.0f;  // Z轴方向
    break;
  case SHD_WAVE_BANDS_DIRECTION_DIAGONAL:
    n = (p.x + p.y + p.z) * 10.0f;  // 对角线方向
    break;
}
```

**数学公式**:
- **X轴**: $$n = p_x \times 20.0$$
- **Y轴**: $$n = p_y \times 20.0$$
- **Z轴**: $$n = p_z \times 20.0$$
- **对角线**: $$n = (p_x + p_y + p_z) \times 10.0$$

```mermaid
    C -->|Z| F[p.z * 20.0]
    C -->|Diagonal| G[(p.x+p.y+p.z) * 10.0]
    D --> H[Wave Profile]

    F --> H

```

### 类型2: Rings (环状波)

**数学原理**: 基于距离的径向波浪

```cpp
// C++实现 (lines 179-196)
case SHD_WAVE_RINGS:
  float3 rp = p;
  switch (rings_direction_) {
    case SHD_WAVE_RINGS_DIRECTION_X:
      rp *= float3(0.0f, 1.0f, 1.0f);  // 忽略X分量
      break;
    case SHD_WAVE_RINGS_DIRECTION_Y:
      rp *= float3(1.0f, 0.0f, 1.0f);  // 忽略Y分量
      break;
    case SHD_WAVE_RINGS_DIRECTION_Z:
      rp *= float3(1.0f, 1.0f, 0.0f);  // 忽略Z分量
      break;
    case SHD_WAVE_RINGS_DIRECTION_SPHERICAL:
      /* Ignore - 使用完整向量 */
      break;
  }
  n = len_v3(rp) * 20.0f;  // 计算模长
  break;
```

**数学公式**:
- **X轴环**: $$n = \sqrt{p_y^2 + p_z^2} \times 20.0$$
- **Y轴环**: $$n = \sqrt{p_x^2 + p_z^2} \times 20.0$$
- **Z轴环**: $$n = \sqrt{p_x^2 + p_y^2} \times 20.0$$
- **球形**: $$n = \sqrt{p_x^2 + p_y^2 + p_z^2} \times 20.0$$

**可视化**:
```mermaid

    C -->|X| D[Ignore X: (0,1,1)]
    C -->|Y| E[Ignore Y: (1,0,1)]
```

## 扭曲算法详解

### 扭曲算法流程

```cpp
// C++实现 (lines 201-206)
if (distortion[i] != 0.0f) {
  n += distortion[i] *
       (noise::perlin_fbm<float3>(p * dscale[i], detail[i], droughness[i], 2.0f, true) *
            2.0f -
        1.0f);
}
```

### 数学分解

**步骤1**: 生成分形布朗运动(FBM)噪声
$$noise = \text{PerlinFBM}(p \times dscale, detail, roughness, 2.0)$$

**步骤2**: 归一化到 `[-1, 1]` 范围
$$noise_{norm} = noise \times 2.0 - 1.0$$

**步骤3**: 应用扭曲强度
$$n_{distorted} = n + distortion \times noise_{norm}$$

### FBM噪声参数

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `p * dscale` | 扭曲空间坐标 | `dscale = 1.0` |
| `detail` | 倍频程数量 | `2.0` |
| `roughness` | 振幅衰减 | `0.5` |
| `2.0f` | Lacunarity (间隙度) | `2.0` |
| `true` | 递归标志 | `true` |

### 扭曲效果可视化

```mermaid
graph LR
    A[原始波形] --> B{Distortion}
    B -->|0.0| C[原始波形]
    B -->|>0.0| D[FBM噪声]
    D --> E[归一化到[-1,1]]
    E --> F[缩放扭曲强度]
    F --> G[叠加到原始波形]
    C --> G
    G --> H[最终波形]
```

- **频率**: $$f = \frac{20.0}{2\pi} \approx 3.18$$

**视觉效果**:
- 控制波浪的密度
- 值越大，波浪越密集
- 默认值提供适中的波浪密度

**出现位置**:
```cpp
// Bands Diagonal

```

**数学意义**:
- **对角线缩放**: 由于对角线长度是轴向的 $\sqrt{3}$ 倍
- **长度补偿**: $\frac{20.0}{\sqrt{3}} \approx 11.55$，使用10.0作为近似值

**计算验证**:
- 轴向距离: $d_{axis} = p_x$
- 对角线距离: $d_{diag} = p_x + p_y + p_z$
- 当 $p_x = p_y = p_z$ 时: $d_{diag} = 3p_x$
- 为了保持相同波长: $3 \times 10.0 = 30.0 \approx 20.0 \times \sqrt{3} \approx 34.6$

## M_PI_2偏移的数学原因

### 问题描述

为什么正弦波剖面使用 `n - M_PI_2` 而不是直接使用 `n`？

### 数学分析

**标准正弦波**:
$$f(x) = \sin(x)$$

**波形特征**:
- $x = 0$ 时，$\sin(0) = 0$
- 波形从零点开始

**Blender的正弦波**:
$$f(x) = 0.5 + 0.5 \times \sin(x - \frac{\pi}{2})$$

**波形特征**:
- $x = 0$ 时，$\sin(-\frac{\pi}{2}) = -1$
- $f(0) = 0.5 + 0.5 \times (-1) = 0$
- 但经过偏移后，$x = \frac{\pi}{2}$ 时，$f(\frac{\pi}{2}) = 0.5 + 0.5 \times \sin(0) = 0.5$

### 实际效果对比

**不加偏移**:
$$f(x) = 0.5 + 0.5 \times \sin(x)$$
- $x = 0$: $f = 0.5$ (中点)
- $x = \frac{\pi}{2}$: $f = 1.0$ (峰值)
- $x = \pi$: $f = 0.5$ (中点)
- $x = \frac{3\pi}{2}$: $f = 0.0$ (谷值)

**加偏移**:
$$f(x) = 0.5 + 0.5 \times \sin(x - \frac{\pi}{2}) = 0.5 - 0.5 \times \cos(x)$$
- $x = 0$: $f = 0.0$ (谷值)
- $x = \frac{\pi}{2}$: $f = 0.5$ (中点)
- $x = \pi$: $f = 1.0$ (峰值)
- $x = \frac{3\pi}{2}$: $f = 0.5$ (中点)

### 为什么选择这种偏移？

**视觉一致性**:
1. **波形起点**: 从谷值(0.0)开始，更符合"波浪从低到高"的直觉
2. **相位对齐**: 与其他剖面(Saw/Tri)的相位保持一致
3. **数学简化**: 可以转换为 `-cos(x)` 形式，计算效率略高

**相位一致性验证**:
- **Sine**: $0.5 - 0.5 \times \cos(x)$
- **Saw**: $x/(2\pi) - \lfloor x/(2\pi) \rfloor$ (从0开始线性上升)
- **Tri**: $2 \times |x/(2\pi) - \lfloor x/(2\pi) + 0.5 \rfloor|$ (从0开始线性上升)

所有剖面在 $x = 0$ 时都输出接近0的值，保持了相位一致性。

## 浮点精度修复

### 问题场景

**问题**: 当坐标值为精确的整数或特殊值时，浮点运算可能产生精度误差

**示例**:
- 输入: $p = (1.0, 0.0, 0.0)$
- 计算: $n = p.x \times 20.0 = 20.0$
- 后续: $val = \sin(20.0 - \frac{\pi}{2})$

### 修复方法

```cpp
p = (p + 0.000001f) * 0.999999f;
```

### 数学原理

**变换分解**:
1. **加小偏移**: $p' = p + 0.000001$
   - 避免 $p = 0.0$ 时的边界情况
   - 避免 $p$ 为精确整数时的问题

2. **乘接近1的数**: $p'' = p' \times 0.999999$
   - 将值略微缩小，避免精确值
   - 保持数值接近原始值

**误差分析**:
- 原始值: $p = 1.0$
- 修正后: $p'' = (1.0 + 0.000001) \times 0.999999 = 1.000000999999$
- 误差: $\approx 10^{-6}$ (可忽略)

### 为什么需要这个修复？

**1. 避免周期性问题**
```glsl
// 如果没有修复
n = 1.0 * 20.0 = 20.0
val = sin(20.0 - M_PI_2)  // 精确值可能导致边界问题

// 修复后
n = 1.000000999999 * 20.0 = 20.00001999998
val = sin(20.00001999998 - M_PI_2)  // 正常计算
```

**2. 避免除零和边界情况**
- 在 `saw` 和 `tri` 剖面中，需要除以 $2\pi$
- 如果 $n$ 是 $2\pi$ 的整数倍，可能导致精确的整数结果
- 修正后避免这些精确边界

**3. 保持数值稳定性**
- 在GPU并行计算中，边界值可能导致不一致
- 修正确保所有线程的计算路径一致

### 实际效果

| 场景 | 无修复 | 有修复 |
|------|--------|--------|
| $p = (0, 0, 0)$ | $n = 0$ | $n \approx 0.00002$ |
| $p = (1, 0, 0)$ | $n = 20$ | $n \approx 20.00002$ |
| $p = (0.5, 0, 0)$ | $n = 10$ | $n \approx 10.00001$ |

**结论**: 修正引入的误差极小($10^{-6}$)，但能有效避免精度问题。
