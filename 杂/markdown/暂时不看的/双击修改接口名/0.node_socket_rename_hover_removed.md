# Node Socket Rename Hover Logic (Removed for Now)

This note preserves the hover-highlight-only code that was removed while keeping the double-click
rename behavior.

## Purpose

The removed code added a stronger hover affordance for editable node socket labels in the node
editor. It did not implement rename entry by itself. Instead, it supported a visual cue and the
small plumbing needed to mark the generated button so the widget drawing code could recognize it.

## Removed Pieces

### 1. Secondary flag in `UI_interface_c.hh`

File: `source/blender/editors/include/UI_interface_c.hh`

```cpp
/** Node editor socket name that should show a stronger hover-only rename highlight. */
BUT2_NODE_SOCKET_RENAME_LABEL = 1 << 2,
```

This flag marked a button as a node socket rename label, so only that specific UI element received
the custom hover treatment.

### 2. Helper declaration in `UI_interface_c.hh`

File: `source/blender/editors/include/UI_interface_c.hh`

```cpp
Button *block_last_but_get(const Block *block);
```

This helper exposed the most recently created button in a block, so code using `layout->prop(...)`
could fetch the generated button pointer afterward and tag it with the flag above.

### 3. Helper definition in `interface_query.cc`

File: `source/blender/editors/interface/interface_query.cc`

```cpp
Button *block_last_but_get(const Block *block)
{
  return block ? block->last_but() : nullptr;
}
```

This was the implementation of the helper declaration above.

### 4. Tagging the generated socket-name button in `drawnode.cc`

File: `source/blender/editors/space_node/drawnode.cc`

```cpp
if (ui::Button *button = ui::block_last_but_get(layout->block())) {
  ui::button_flag2_enable(button, ui::BUT2_NODE_SOCKET_RENAME_LABEL);
}
```

This ran immediately after the socket name `layout->prop(...)` call and attached the specialized
flag to that newly created button.

### 5. Hover drawing block in `interface_widgets.cc`

File: `source/blender/editors/interface/interface_widgets.cc`

```cpp
if ((but->flag2 & BUT2_NODE_SOCKET_RENAME_LABEL) && (state->but_flag & UI_HOVER) &&
    !(state->but_flag & UI_SELECT))
{
  bTheme *btheme = theme::theme_get();
  uiWidgetColors hover_color = btheme->tui.wcol_list_item;
  WidgetBase wtb;
  widget_init(&wtb);

  const int x_pad = int(0.15f * BLI_rcti_size_y(rect));
  rcti hover_rect = *rect;
  hover_rect.xmin -= x_pad;
  hover_rect.xmax += x_pad;

  const float rad = widget_radius_from_zoom(zoom, &hover_color);
  round_box_edges(&wtb, roundboxalign, &hover_rect, rad);

  color_blend_v3_v3(hover_color.inner, hover_color.text, 0.2f);
  hover_color.inner[3] = 50;
  hover_color.outline[3] = 255;
  widgetbase_draw(&wtb, &hover_color);
}
```

This was the actual hover-only highlight rendering. It:

- Checked for the specialized button flag.
- Required hover state and skipped selected state.
- Expanded the draw rect slightly in X to make the target feel less razor-thin.
- Reused theme list-item colors, then blended them toward text color for a subtle highlight.
- Drew a rounded translucent background with a fully visible outline.

## What Was Intentionally Kept

The double-click rename path remains in the codebase. The interaction changes in
`source/blender/editors/interface/interface_handlers.cc` were not part of this removal.

## Restore Checklist

If we want the hover highlight back later, restore all of the following together:

1. Re-add `BUT2_NODE_SOCKET_RENAME_LABEL` in `UI_interface_c.hh`.
2. Re-add `block_last_but_get(...)` declaration and definition.
3. Re-add the `button_flag2_enable(...)` block in `drawnode.cc` after the socket name property is
   created.
4. Re-add the hover drawing block in `interface_widgets.cc`.

Restoring only the widget drawing block is not enough, because no button will be tagged.
