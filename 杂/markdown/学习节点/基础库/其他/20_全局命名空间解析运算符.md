# C++ `::` 全局命名空间解析运算符详解

> 解释 `::类名` 的用法，以及 `::` 在 C++ 中的各种使用场景。

---

## 1. `::` 的基本含义

`::` 在 C++ 中称为 **"作用域解析运算符"（Scope Resolution Operator）**。

当 `::` 前面**没有任何命名空间名称**时，它特指 **全局命名空间（Global Namespace）**。

```cpp
// :: 表示"全局命名空间"
::Foo  // 全局命名空间中的 Foo

// 对比：
namespace bke {
    class Foo { };  // bke::Foo
}

class Foo { };     // ::Foo（全局命名空间）
```

---

## 2. `::类名` 的用法

### 场景 1：明确指定全局命名空间中的类型

```cpp
// 全局命名空间中定义了一个类
class CurvesGeometry {
    // ...
};

namespace bke {
    // bke 命名空间中也定义了一个同名类
    class CurvesGeometry {
        // ...
    };
    
    void foo() {
        CurvesGeometry cg;      // 使用 bke::CurvesGeometry（当前命名空间）
        ::CurvesGeometry cg2;   // 使用全局的 ::CurvesGeometry
    }
}
```

**为什么要用 `::`？**

当内部命名空间定义了与外部同名的类型时，内部类型会**隐藏**外部类型：

```cpp
// 全局命名空间
class Foo { int x; };

namespace inner {
    class Foo { int y; };  // 隐藏了全局的 ::Foo
    
    void test() {
        Foo f1;      // 使用 inner::Foo（当前命名空间）
        ::Foo f2;    // 使用全局的 ::Foo（必须加 ::）
    }
}
```

### 场景 2：在命名空间中使用其他命名空间的类型

```cpp
namespace blender {

namespace gpu {
class Texture;  // 在 gpu 命名空间中定义 Texture
}

namespace seq {

// 在 seq 命名空间中使用 gpu 命名空间的 Texture
// 需要加 gpu:: 前缀
gpu::Texture *preview_cache_get_gpu_texture(
    Scene *scene, int timeline_frame, int display_channel, int width, int height);

} // namespace seq
} // namespace blender
```

**注意：这里用的是 `gpu::Texture`，不是 `::Texture`**

### 场景 3：访问全局函数（特别是 C API）

```cpp
namespace blender {

void CloseHandle(void *handle);  // Blender 自己的函数

void foo() {
    // 调用 Windows API 的 CloseHandle
    ::CloseHandle(handle);  // 明确指定全局命名空间的 CloseHandle
    
    // 如果不加 ::，会调用 blender::CloseHandle
    CloseHandle(handle);  // 调用 blender::CloseHandle
}

} // namespace blender
```

**在 Blender 中的实际例子：**

```cpp
// source/blender/blenlib/intern/BLI_mmap.cc:260
::GetProcAddress(kernelbase, "MapViewOfFile3")

// 明确调用 Windows API 的全局函数 GetProcAddress
// 而不是某个命名空间中的同名函数
```

### 场景 4：访问全局类型的成员（特别是 Objective-C/C++）

```cpp
// source/blender/gpu/metal/mtl_shader.hh:151
::MTLRenderPipelineDescriptor *pso_descriptor_ = nil;

// 在 Objective-C++ 中，::MTLRenderPipelineDescriptor 表示全局命名空间中的 Metal 类型
// 避免与当前命名空间中的同名类型冲突
```

## 3. `::` 的各种用法总结

| 用法 | 含义 | 示例 |
|------|------|------|
| `::Name` | 全局命名空间中的 Name | `::CurvesGeometry` |
| `ns::Name` | 命名空间 ns 中的 Name | `bke::CurvesGeometry` |
| `Class::member` | 类成员访问 | `CurvesGeometry::wrap()` |
| `ns1::ns2::Name` | 嵌套命名空间 | `blender::bke::CurvesGeometry` |

### 命名空间查找规则

```cpp
namespace blender {
namespace bke {

class CurvesGeometry { };

void foo() {
    // 编译器查找 CurvesGeometry 的顺序：
    // 1. 当前命名空间：bke::CurvesGeometry ✅ 找到！
    CurvesGeometry cg1;
    
    // 2. 明确指定全局命名空间
    ::CurvesGeometry cg2;  // 全局的 CurvesGeometry
    
    // 3. 明确指定其他命名空间
    blender::bke::CurvesGeometry cg3;  // 完整路径
}

} // namespace bke
} // namespace blender
```

## 4. Blender 中的实际使用

### 例子 1：DNA 结构体和 BKE 类的区分

```cpp
// DNA_curves_types.h
namespace blender {
struct CurvesGeometry {
    // DNA 版本（在 blender 命名空间中）
};
} // namespace blender

// BKE_curves.hh
namespace blender {
namespace bke {

class CurvesGeometry : public blender::CurvesGeometry {
    // BKE 版本（在 blender::bke 命名空间中）
};

} // namespace bke
} // namespace blender

// 使用时：
blender::CurvesGeometry dna;      // DNA 版本
blender::bke::CurvesGeometry bke; // BKE 版本
```

**注意：这里没有用到 `::`，因为类型不在全局命名空间中**

### 例子 2：前向声明中的用法

```cpp
// DNA_curves_types.h
namespace blender {

// 前向声明
namespace bke {
class CurvesGeometry;           // bke::CurvesGeometry
class CurvesGeometryRuntime;    // bke::CurvesGeometryRuntime
}

// 使用完整路径
struct CurvesGeometry {
    bke::CurvesGeometry &wrap();  // 返回 bke::CurvesGeometry&
};

} // namespace blender
```

### 例子 3：函数定义中的用法

```cpp
// 在 BKE_curves.hh 文件末尾
namespace blender {

// 给 blender::CurvesGeometry（不是 bke::CurvesGeometry）定义方法
inline bke::CurvesGeometry &CurvesGeometry::wrap()
{
    return *reinterpret_cast<bke::CurvesGeometry *>(this);
}

} // namespace blender
```

**这里 `CurvesGeometry::wrap()` 指的是 `blender::CurvesGeometry::wrap()`**

## 5. 常见误区

### 误区 1：`::` 表示"全局命名空间"，但类型可能在其他命名空间

```cpp
// ❌ 错误理解：
::CurvesGeometry  // 以为这是全局命名空间的 CurvesGeometry

// ✅ 正确理解：
// 实际上 CurvesGeometry 定义在 blender 命名空间中
// 所以完整名是 blender::CurvesGeometry，不是 ::CurvesGeometry
```

### 误区 2：在命名空间内部使用 `::` 访问同命名空间的类型

```cpp
namespace blender {

struct CurvesGeometry { };

void foo() {
    // ❌ 不必要：
    ::CurvesGeometry cg;  // 错误！全局命名空间中没有 CurvesGeometry
    
    // ✅ 正确：
    CurvesGeometry cg;  // 直接使用，编译器会在当前命名空间查找
}

} // namespace blender
```

### 误区 3：混淆 `::` 和 `namespace::`

```cpp
namespace blender {
namespace bke {

class CurvesGeometry { };

void foo() {
    // ✅ 正确：使用当前命名空间的类型
    CurvesGeometry cg1;
    
    // ✅ 正确：使用完整路径
    blender::bke::CurvesGeometry cg2;
    
    // ❌ 错误：全局命名空间中没有 CurvesGeometry
    ::CurvesGeometry cg3;
}

} // namespace bke
} // namespace blender
```

## ✅ 总结

| 问题 | 答案 |
|------|------|
| `::类名` 是什么？ | 全局命名空间中的类 |
| 什么时候用 `::类名`？ | 当需要明确指定全局命名空间，避免命名冲突时 |
| Blender 中常用吗？ | 不常用，因为 Blender 的类型大多在 `blender` 或 `blender::bke` 命名空间中 |
| 和 `namespace::类名` 的区别？ | `::类名` 是全局的，`namespace::类名` 是指定命名空间的 |

**关键原则：**
1. `::Name` = 全局命名空间中的 Name
2. `ns::Name` = 命名空间 ns 中的 Name
3. 在命名空间内部，直接使用 `Name` 即可（编译器会优先在当前命名空间查找）
4. 只有发生命名冲突时，才需要用 `::` 或 `ns::` 明确指定
