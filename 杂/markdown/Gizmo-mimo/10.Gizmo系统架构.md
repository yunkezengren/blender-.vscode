## 1. 概述

Blender 的 gizmo 系统是一个多层次的架构，提供了灵活的交互工具框架。理解其架构对于创建和扩展 gizmo 至关重要。

<mark style="background-color: #1976D2; color: white; padding: 2px 6px; border-radius: 3px;">★ Key Point</mark>
Gizmo 系统的核心是**分离关注点**：类型定义（API）、实例管理（Group）、空间管理（Map）和渲染（Draw）。

## 2. 核心数据结构

### 2.1. wmGizmo

**源码位置**: `source/blender/windowmanager/gizmo/WM_gizmo_types.hh:45-120`

```cpp
struct wmGizmo {
  wmGizmoType *type;                    // 类型定义
  struct wmGizmoGroup *parent_gzgroup;  // 所属 Group

  /* 变换矩阵 */
  float matrix_offset[4][4];            // 局部偏移矩阵
  float matrix_space[4][4];             // 空间变换矩阵

  /* 状态 */
  int flag;                             // 标志位
  int state;                            // 状态位（高亮、选中等）
  int highlight_part;                   // 当前高亮的部件 ID

  /* 属性 */
  PointerRNA ptr;                       // RNA 属性指针
  wmGizmoProperty *properties;          // 目标属性绑定

  /* 交互 */
  void *interaction_data;               // 交互数据
  wmGizmoFnModal custom_modal;          // 自定义模态

  /* 外观 */
  float scale_basis;                    // 基础缩放
  float line_width;                     // 线宽
  float color[4];                       // 颜色
};
```

**关键字段说明**:
- `matrix_offset`: gizmo 相对于其空间的局部变换
- `matrix_space`: 将 gizmo 从局部空间转换到屏幕空间
- `highlight_part`: 用于区分点击的是哪个部件（如边缘、角、中心）
- `properties`: 双向绑定到外部数据源

### 2.2. wmGizmoType

**源码位置**: `source/blender/windowmanager/gizmo/WM_gizmo_types.hh:122-160`

```cpp
struct wmGizmoType {
  char idname[64];                      // 标识符（如 "GIZMO_GT_cage_2d"）

  /* 回调函数 */
  wmGizmoFnDraw draw;                   // 绘制
  wmGizmoFnDrawSelect draw_select;      // 选择绘制
  wmGizmoFnTestSelect test_select;      // 点击测试
  wmGizmoFnSetup setup;                 // 初始化
  wmGizmoFnInvoke invoke;               // 开始交互
  wmGizmoFnModal modal;                 // 模态处理
  wmGizmoFnExit exit;                   // 结束交互
  wmGizmoFnPropertyUpdate property_update; // 属性更新
  wmGizmoFnCursorGet cursor_get;        // 光标样式

  /* 其他 */

```

**设计模式**: 这是一个典型的**策略模式**，通过函数指针允许不同类型实现不同的行为。

### 2.3. wmGizmoGroup

**源码位置**: `source/blender/windowmanager/gizmo/WM_gizmo_types.hh:162-200`

```cpp
struct wmGizmoGroup {
  wmGizmoGroupType *type;               // Group 类型定义
  wmGizmoMap *gzmap;                    // 所属 Map

  /* Gizmo 列表 */
  ListBase gizmos;                      // 包含的 gizmo 实例
```

**作用**: Group 是 gizmo 的**容器和管理者**，负责：
- 组织多个相关的 gizmo

## 4. 生命周期

### 4.1. 创建阶段

```mermaid
graph TD
    A[WM_gizmomaptype_ensure] --> B[创建 Map Type]
    B --> C[WM_gizmogrouptype_append_and_link]
    C --> D[创建 Group Type]
    D --> E[用户交互触发]
    E --> F[Group poll() 检查]
    F -->|通过| G[Group setup()]
    F -->|失败| H[不创建]
    G --> I[WM_gizmo_new()]
    I --> J[gizmo setup()]
    J --> K[添加到 Group]
```

**时序**:
1. **编辑器初始化**: 注册 Map 和 Group 类型
2. **用户操作**: 触发 gizmo 显示条件
3. **Poll 检查**: 确定是否显示
4. **Setup**: 创建 gizmo 实例并初始化
5. **Refresh**: 更新状态和属性绑定

### 4.2. 交互阶段

```mermaid
graph TD
    A[用户点击] --> B[测试选择 test_select]
    B --> C[返回部件 ID]
    C --> D[设置 highlight_part]
    D --> E[调用 invoke()]
    E --> F[分配交互数据]
    F --> G[进入模态]
    G --> H[事件循环]
    H --> I[modal() 处理]
    I --> J[更新矩阵/属性]
    J --> K[标记重绘]
    K --> L[draw() 渲染]
    L --> H
```

**事件流**:
```
鼠标按下 → test_select() → 确定部件 → invoke() → modal() 循环 → exit()
```

### 4.3. 销毁阶段

**源码位置**: `source/blender/windowmanager/gizmo/intern/wm_gizmo.cc:200-250`

```cpp
void WM_gizmo_free(wmGizmo *gz)
{
  /* 1. 释放交互数据 */
  if (gz->interaction_data) {
    MEM_freeN(gz->interaction_data);
  }

  /* 2. 释放属性绑定 */
  wmGizmoProperty *prop = gz->properties;
  while (prop) {
    wmGizmoProperty *next = prop->next;
    MEM_freeN(prop);
    prop = next;
  }

  /* 3. 释放 gizmo 本身 */
  MEM_freeN(gz);
}
```

**Group 释放**:
```cpp
void WM_gizmo_group_free(wmGizmoGroup *gzgroup)
{
  /* 释放所有 gizmo */
  LISTBASE_FOREACH_MUTABLE(wmGizmo *, gz, &gzgroup->gizmos) {
    WM_gizmo_free(gz);
  }

  /* 释放自定义数据 */
  if (gzgroup->customdata && gzgroup->customdata_free) {
    gzgroup->customdata_free(gzgroup->customdata);
  }

  MEM_freeN(gzgroup);
}
```

## 5. 事件处理流程

### 5.1. 事件路由

**源码位置**: `source/blender/windowmanager/gizmo/intern/wm_gizmo_event.cc:50-150`

```cpp
int WM_gizmo_modal(bContext *C, wmEvent *event, void * /*op*/)
{
  wmGizmoMap *gzmap = CTX_wm_gizmo_map(C);
  if (!gzmap) {
    return OPERATOR_PASS_THROUGH;
  }

  /* 1. 查找活跃 gizmo */
  wmGizmo *active_gz = gzmap->active_gizmo;

  if (active_gz) {
    /* 2. 有活跃 gizmo，直接发送到它 */
    return WM_gizmo_modal_to_active_gizmo(C, event, active_gz);
  }

  /* 3. 没有活跃 gizmo，测试选择 */
  wmGizmo *gz = WM_gizmo_find_from_event(C, event);

  if (gz) {
    /* 4. 找到 gizmo，调用其 invoke */
    return WM_gizmo_invoke_from_event(C, event, gz);
  }

  return OPERATOR_PASS_THROUGH;
}
```

### 5.2. 模态操作

**源码位置**: `source/blender/windowmanager/gizmo/intern/wm_gizmo_group.cc:479-562`

```cpp
static wmOperatorStatus gizmo_tweak_modal(bContext *C, wmOperator *op, const wmEvent *event)
{
  GizmoTweakData *mtweak = (GizmoTweakData *)op->customdata;
  wmGizmo *gz = mtweak->gz_modal;

  /* 1. 处理修饰键事件 */
  if (event->type == EVT_MODAL_MAP) {
    switch (event->val) {
      case TWEAK_MODAL_PRECISION_ON:
        mtweak->flag |= WM_GIZMO_TWEAK_PRECISE;
        break;
      case TWEAK_MODAL_SNAP_ON:
        mtweak->flag |= WM_GIZMO_TWEAK_SNAP;
        break;
      case TWEAK_MODAL_PRECISION_OFF:
        mtweak->flag &= ~WM_GIZMO_TWEAK_PRECISE;
        break;
      case TWEAK_MODAL_SNAP_OFF:
        mtweak->flag &= ~WM_GIZMO_TWEAK_SNAP;
        break;
    }
  }

  /* 2. 调用 gizmo 的 modal 回调 */
  wmGizmoFnModal modal_fn = gz->custom_modal ? gz->custom_modal : gz->type->modal;
  if (modal_fn) {
    wmOperatorStatus ret = modal_fn(C, gz, event, (eWM_GizmoFlagTweak)mtweak->flag);

    /* 3. 处理返回值 */
    if (ret != OPERATOR_RUNNING_MODAL) {
      /* 结束交互 */
      WM_gizmo_exit(C, gz, ret == OPERATOR_CANCELLED);
      WM_event_remove_modal_handler(op);
    }
  }

  return OPERATOR_PASS_THROUGH;
}
```

**完整事件流**:
```
1. 用户按下鼠标
2. WM_gizmo_modal() 被调用
3. 查找或测试选择 gizmo
4. 调用 gizmo->invoke()
5. 创建 GizmoTweakData
6. 注册为模态操作
7. 后续事件发送到 gizmo_tweak_modal()
8. 处理修饰键变化
9. 调用 gizmo->modal()
10. 用户释放鼠标或按确认/取消
11. 调用 gizmo->exit()
12. 清理并退出模态
```

## 6. 渲染管线

### 6.1. 绘制流程

**源码位置**: `source/blender/windowmanager/gizmo/intern/wm_gizmo_draw.cc:100-200`

```cpp
void WM_gizmo_draw(const bContext *C, wmGizmo *gz)
{
  /* 1. 检查可见性 */
  if (gz->flag & WM_GIZMO_HIDDEN) {
    return;
  }

  /* 2. 推送矩阵 */
  GPU_matrix_push();
  GPU_matrix_mul(gz->matrix_space);
  GPU_matrix_mul(gz->matrix_offset);

  /* 3. 设置状态 */
  if (gz->state & WM_GIZMO_STATE_HIGHLIGHT) {
    /* 高亮状态 */
    float color[4];
    gizmo_color_get(gz, true, color);
    immUniformColor4fv(color);
  }

  /* 4. 调用类型绘制函数 */
  if (gz->type->draw) {
    gz->type->draw(C, gz);
  }

  /* 5. 恢复矩阵 */
  GPU_matrix_pop();
}
```

**Group 绘制**:
```cpp
void WM_gizmo_group_draw(const bContext *C, wmGizmoGroup *gzgroup)
{
  /* 1. 调用 draw_prepare */
  if (gzgroup->type->draw_prepare) {
    gzgroup->type->draw_prepare(C, gzgroup);
  }

  /* 2. 绘制所有 gizmo */
  LISTBASE_FOREACH(wmGizmo *, gz, &gzgroup->gizmos) {
    WM_gizmo_draw(C, gz);
  }
}
```

## 8. 空间变换

### 8.1. 矩阵系统

**三种矩阵**:

1. **matrix_offset** (局部偏移)
   - 描述: gizmo 相对于其中心的变换
   - 用途: 存储 gizmo 的位置、旋转、缩放
   - 示例: 平移时修改 `matrix_offset[3][0/1]`

2. **matrix_space** (空间变换)
   - 描述: 从 gizmo 局部空间到屏幕空间的转换
   - 用途: 将 gizmo 放置在正确位置
   - 示例: 节点编辑器的缩放和偏移

3. **最终矩阵** = `matrix_space` × `matrix_offset`

**源码位置**: `source/blender/windowmanager/gizmo/intern/wm_gizmo_util.cc:50-100`

```cpp
void WM_gizmo_calc_matrix_final(const wmGizmo *gz, float matrix_final[4][4])
{
  mul_m4_m4m4(matrix_final, gz->matrix_space, gz->matrix_offset);
}
```

### 8.2. 坐标投影

**源码位置**: `source/blender/editors/gizmo_library/gizmo_library_intern.hh:100-150`

```cpp
bool gizmo_window_project_2d(const bContext *C,
                             wmGizmo *gz,
                             const float2 &mval,
                             int /*offset*/,
                             bool use_matrix_space,
                             float r_point_local[2])
{
  /* 1. 获取最终矩阵 */
  float matrix_final[4][4];
  WM_gizmo_calc_matrix_final(gz, matrix_final);

  /* 2. 反转矩阵 */
  float matrix_inv[4][4];
  invert_m4_m4(matrix_inv, matrix_final);

  /* 3. 转换鼠标坐标 */
  float vec[4] = {mval.x, mval.y, 0.0f, 1.0f};
  mul_m4_v4(matrix_inv, vec);

  r_point_local[0] = vec[0];
  r_point_local[1] = vec[1];

  return true;
}
```

**投影流程**:
```
屏幕坐标 (mval)
    ↓
乘以 matrix_final 的逆矩阵
    ↓
局部坐标 (point_local)
```

## 10. 总结

### 10.1. 架构层次

```
编辑器层 (Editor)
  ↓ 注册
Gizmo Map (空间容器)
  ↓ 包含
Gizmo Group (逻辑容器)
  ↓ 包含
Gizmo (交互单元)
  ↓ 类型
wmGizmoType (行为定义)
```

### 10.2. 关键设计模式

| 模式 | 应用 | 优势 |
|------|------|------|
| **策略模式** | wmGizmoType 回调 | 灵活的行为扩展 |
| **组合模式** | Group 包含多个 gizmo | 统一管理 |
| **观察者模式** | 属性绑定 | 自动同步 |
| **状态模式** | 交互生命周期 | 清晰的状态转换 |

### 10.3. 数据流

```
用户输入 → 事件系统 → 键映射 → 模态操作 → gizmo.modal() → 属性更新 → 重绘
```

### 10.4. 扩展点

1. **创建新类型**: 实现 wmGizmoType 回调

4. **扩展修饰键**: 修改 tweak_flag 或直接检查事件
5. **空间变换**: 自定义 matrix_space 计算
