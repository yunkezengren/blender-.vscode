# 技术文档格式规范

## Markdown 文档结构模板

### 1. 文档头部格式
```markdown
# [文档标题]

## 目录

- [概述](#概述)
- [核心组件分析](#核心组件分析)
  - [1. 组件名称](#1-组件名称)
  - [2. 组件名称](#2-组件名称)
- [核心函数解析](#核心函数解析)
  - [1. 函数名称](#1-函数名称)
  - [2. 函数名称](#2-函数名称)
- [实现细节](#实现细节)
- [总结](#总结)
```

### 2. 章节格式规范
- **二级标题后必须添加目录链接**: `## 章节名称  [[⬆](#目录)]`
- **三级标题**: `### 1. 编号 + 标题`
- **四级标题**: `#### 2.1 编号 + 标题`

### 3. 代码块引用格式
- **每个涉及代码的地方都必须标注位置**: 
  ```markdown
  **定义位置**: `文件名:行号范围`
  ```
- **示例**: `**定义位置**: `draw_manager_text.cc:54-69``

### 4. 代码展示格式
```cpp
// 代码块前要有位置标注
**定义位置**: `文件名:行号范围`

struct Example {
    int member;
};
```

### 5. 内容组织原则
- **概述**: 简要介绍系统功能和目的
- **核心组件**: 按重要性排序，每个组件包含：
  - 定义位置
  - 功能说明
  - 关键特性
- **核心函数**: 按调用关系或重要性排序
- **实现细节**: 深入技术实现，包含：
  - 内存管理
  - 性能优化
  - 错误处理
  - 算法分析
- **总结**: 归纳设计原则和优秀实践

### 6. 技术分析要点
- **内存管理策略**: 柔性数组、内存池、缓存友好设计
- **性能优化技术**: 并行处理、状态缓存、批量操作
- **坐标系统处理**: 多坐标系支持、投影变换
- **错误处理**: 边界检查、断言使用、异常处理
- **设计模式**: 模板元编程、RAII、工厂模式等

### 7. 文档质量要求
- **准确性**: 代码位置必须准确，行号范围要精确
- **完整性**: 每个重要组件都要有详细分析
- **可读性**: 使用清晰的技术术语，避免模糊表述
- **实用性**: 提供实际代码示例和使用场景
- **深度**: 不仅要说明"是什么"，还要解释"为什么"和"怎么做"

### 8. 特殊格式要求
- **强调重要概念**: 使用**粗体**标记关键术语
- **代码注释**: 在代码块中添加必要的解释注释
- **数学公式**: 复杂算法要提供数学表达式
- **流程图**: 复杂流程可用文字描述步骤

### 9. 链接规范
- **内部链接**: 使用锚点格式 `[#标题]`
- **返回目录**: 每个主要章节后添加 `[[⬆](#目录)]`
- **代码引用**: 使用反引号格式 `文件名:行号`

### 10. 文档维护
- **及时更新**: 代码变更后同步更新文档
- **版本控制**: 重要修改要记录版本信息
- **交叉引用**: 相关文档要互相引用
